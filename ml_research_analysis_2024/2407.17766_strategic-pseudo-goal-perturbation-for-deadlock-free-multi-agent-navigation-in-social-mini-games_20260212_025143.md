# Strategic Pseudo-Goal Perturbation for Deadlock-Free Multi-Agent Navigation in Social Mini-Games

## Quick Facts
- arXiv ID: 2407.17766
- Source URL: https://arxiv.org/abs/2407.17766

## Executive Summary
This paper introduces Strategic Pseudo-Goal Perturbation (SPGP), a method for resolving deadlocks in multi-agent navigation within social mini-games. The approach combines Safety Barrier Certificates with a strategic perturbation mechanism that generates pseudo goals to guide agents out of deadlock situations. When a deadlock is detected, agents are directed toward temporary waypoints within a predefined radius, which helps break symmetry and resolve conflicts. The method then allows agents to resume their original paths once the deadlock is cleared. Experiments in four social navigation scenarios—doorway, intersection, hallway, and L-corner—show that SPGP achieves 100% success rate in reaching goals without deadlock or collision, outperforming existing methods like SBC, ORCA, and NH-TTC.

## Method Summary
The SPGP method integrates Safety Barrier Certificates (SBCs) with a strategic perturbation mechanism for deadlock resolution. The approach uses quadratic programming to optimize control inputs while enforcing safety constraints. When deadlock is detected (when actual control inputs approach zero while nominal controls remain non-zero), the algorithm generates pseudo-goals within a circular region around each agent's position. These temporary waypoints help break symmetry in the environment and allow agents to escape deadlock situations. The method then allows agents to resume their original paths once the deadlock is cleared. The framework was tested in four social navigation scenarios: doorway, intersection, hallway, and L-corner.

## Key Results
- Achieved 100% success rate in reaching goals without deadlock or collision across all tested scenarios
- Outperformed existing methods (SBC, ORCA, NH-TTC) in terms of success rate and navigational efficiency
- Minimized average velocity changes and path deviations compared to baseline methods
- Demonstrated effectiveness in constrained multi-agent environments through systematic experiments

## Architecture Onboarding

- Component map: Deadlock detection module -> Pseudo-goal selection module -> Safety Barrier Certificate module -> QP optimizer -> Navigation controller
- Critical path:
  1. Monitor actual and nominal control inputs
  2. Detect deadlock condition
  3. Generate pseudo-goal within perturbation radius
  4. Update control inputs to navigate toward pseudo-goal
  5. Resume original path after reaching pseudo-goal
  6. Enforce safety constraints via SBCs throughout
- Design tradeoffs:
  - Perturbation radius vs. path efficiency: Larger radii may resolve deadlocks more effectively but increase path deviation
  - Deadlock detection threshold vs. responsiveness: Lower thresholds may detect deadlocks earlier but increase false positives
  - Safety radius vs. navigation flexibility: Larger safety radii increase safety but reduce navigable space
- Failure signatures:
  - Persistent deadlocks despite pseudo-goal perturbation
  - Excessive path deviation leading to inefficient navigation
  - Collisions occurring due to insufficient safety constraint enforcement
  - Oscillations between deadlock and resolution states
- First 3 experiments:
  1. Implement deadlock detection and test with simple symmetric scenarios (e.g., two agents facing each other in a narrow corridor)
  2. Add pseudo-goal perturbation and evaluate deadlock resolution in the same scenarios
  3. Integrate Safety Barrier Certificates and test for collision-free navigation during deadlock resolution
