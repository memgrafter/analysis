# Enhancing Sampling Protocol for Point Cloud Classification Against Corruptions

## Quick Facts
- arXiv ID: 2408.12062
- Source URL: https://arxiv.org/abs/2408.12062
- Authors: Chongshou Li; Pin Tang; Xinke Li; Yuheng Liu; Tianrui Li
- Reference count: 11
- One-line primary result: PointSP significantly improves robustness against point cloud corruptions, reducing mean error rates by approximately 10% across multiple corruption types

## Executive Summary
This paper addresses the vulnerability of standard point cloud sampling protocols, such as Farthest Point Sampling (FPS), to real-world corruptions like sensor noise and occlusions, which can lead to significant performance drops in critical applications such as autonomous driving. The authors propose PointSP, an enhanced, learning-free sampling protocol that improves robustness against point cloud corruptions. PointSP introduces three key techniques: point reweighting based on isolation rate to mitigate outlier sensitivity, local-global balanced downsampling for scalable and adaptive sampling, and a lightweight tangent plane interpolation method to preserve local geometry while enhancing point cloud density. Extensive experiments on synthetic and real-world corrupted datasets (ModelNet40-C, PointCloud-C, and OmniObject-C) demonstrate that PointSP significantly improves the robustness and accuracy of point cloud classification, outperforming state-of-the-art methods.

## Method Summary
PointSP is an enhanced, learning-free sampling protocol designed to improve the robustness of point cloud classification against real-world corruptions. It introduces three key techniques: point reweighting based on isolation rate to reduce outlier sensitivity, local-global balanced downsampling for adaptive sampling, and tangent plane interpolation for geometry-preserving upsampling. The protocol is architecture-agnostic, requiring no extra learning or modification to the network, enabling seamless integration into existing pipelines. PointSP replaces standard FPS and FSS with its enhanced components, using isolation rates for point weighting, stochastic neighborhood sizing for downsampling, and tangent plane interpolation for upsampling.

## Key Results
- PointSP significantly improves robustness against point cloud corruptions, reducing mean error rates by approximately 10% across multiple corruption types
- The most substantial improvement observed in PointCloud-C dataset compared to ModelNet40-C
- PointSP outperforms state-of-the-art methods while maintaining architecture-agnostic compatibility

## Why This Works (Mechanism)

### Mechanism 1
- Claim: PointSP's point reweighting based on isolation rate reduces outlier selection in key point sampling.
- Mechanism: Isolation rate quantifies how isolated a point is from its k nearest neighbors. Points with higher isolation rates (outliers) are given lower sampling weights, reducing their chance of being selected as key points.
- Core assumption: Outliers tend to be more isolated from local neighborhoods than inliers.
- Evidence anchors:
  - [abstract] "PointSP incorporates key point reweighting to mitigate outlier sensitivity and ensure the selection of representative points."
  - [section] "The point weight is named as isolation rate evaluating the extent of local isolation for a point."
- Break condition: If corruptions create isolated clusters of valid points, this method may mistakenly downweight them.

### Mechanism 2
- Claim: Local-global balanced downsampling adapts to corruption-induced size variations in point clouds.
- Mechanism: By stochastically choosing the neighborhood size k̃ for point removal, the protocol flexibly transitions between local (concentrated patch removal) and global (random sampling) downsampling.
- Core assumption: Real-world corruptions produce variable-sized point clouds, and a single downsampling strategy cannot optimally handle all cases.
- Evidence anchors:
  - [abstract] "It also introduces a local-global balanced downsampling strategy, which allows for scalable and adaptive sampling while maintaining geometric consistency."
  - [section] "This strategy offers a continuous spectrum between local and global downsampling... This flexibility helps simulate various real-world point cloud corruptions."
- Break condition: If k̃ sampling is poorly tuned, it might remove too many points locally or globally, harming downstream performance.

### Mechanism 3
- Claim: Tangent plane interpolation-based upsampling preserves local geometry while restoring point cloud density.
- Mechanism: New points are interpolated between a point and a randomly chosen neighbor along their shared tangent plane, maintaining surface normals.
- Core assumption: Corruptions often reduce point density without significantly altering local surface geometry.
- Evidence anchors:
  - [abstract] "a lightweight tangent plane interpolation method to preserve local geometry while enhancing the density of the point cloud."
  - [section] "This approach ensures the preservation of local geometric features, particularly surface normals, while increasing point density."
- Break condition: If the corruption introduces complex surface discontinuities, tangent plane interpolation might generate points that violate true surface geometry.

## Architecture Onboarding

- Component map:
  - PointSP protocol (replaces standard FPS + FSS)
    - Isolation rate calculator
    - Point reweighting module
    - Local-global balanced downsampler
    - Tangent plane interpolator
    - FFPS key point selector
- Critical path:
  1. Compute isolation rates for all points.
  2. Apply reweighting to generate sampling weights.
  3. Perform local-global balanced downsampling (training) or conditional upsampling (inference).
  4. Select key points using FFPS or stochastic weighted sampling.
- Design tradeoffs:
  - Isolation rate vs. other outlier detection methods: simpler, but may misclassify isolated valid points.
  - Local-global balance vs. fixed neighborhood size: more robust, but adds stochasticity.
  - Tangent plane vs. other upsampling: better geometry preservation, but requires normals.
- Failure signatures:
  - Degraded accuracy when input point clouds are extremely sparse.
  - Over-smoothing when upsampling very noisy surfaces.
  - Performance drop if k̃ sampling is poorly tuned.
- First 3 experiments:
  1. Run PointSP on a clean ModelNet40 test set and compare classification accuracy to baseline FPS.
  2. Apply PointSP to a corrupted ModelNet40-C subset and measure error rate reduction.
  3. Visualize isolation rates and upsampled point distributions for qualitative inspection.
