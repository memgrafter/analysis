# SAKA: An Intelligent Platform for Semi-automated Knowledge Graph Construction and Application

## Quick Facts
- **arXiv ID**: 2410.08094
- **Source URL**: https://arxiv.org/abs/2410.08094
- **Reference count**: 35
- **Primary result**: SAKA enables semi-automated KG construction from structured data and audio, achieving 97.42% VAD accuracy, 10.58% EER for SD, and 74.18% F1 for medical information extraction.

## Executive Summary
SAKA is an intelligent platform designed to address the challenges of manually constructing knowledge graphs (KGs) and transforming audio data into structured knowledge. The platform enables users to semi-automatically construct KGs from structured JSON data by defining entity types and relationships, which are then mapped to the data using Neo4j Cypher statements. Additionally, SAKA introduces an Audio-based KG Information Extraction (AGIE) method that converts audio data into KGs through a pipeline of Voice Activity Detection (VAD), Speaker Diarization (SD), and Medical Information Extractor (MIE) models. The platform also implements a semantic parsing-based knowledge base question answering (KBQA) system that allows users to query the constructed KGs using natural language questions.

## Method Summary
SAKA provides a comprehensive framework for KG construction and application through three main components: (1) semi-automated KG construction from structured JSON data by allowing users to define ontology elements (entity types, relationships, attributes) which are automatically mapped to data and stored in Neo4j, (2) audio-based KG extraction using AGIE method that preprocesses audio with VAD and SD to identify speakers and segments, converts segments to text, and extracts entities and relations using MIE model, and (3) semantic parsing-based KBQA system that translates natural language questions into Cypher queries against user-constructed KGs. The platform was evaluated using medical data, demonstrating effectiveness across all components with specific performance metrics for each module.

## Key Results
- VAD achieved 97.42% accuracy in speech segmentation
- Speaker Diarization reached 10.58% EER in identifying speakers
- MIE model outperformed baselines with 74.18% F1 score on medical information extraction
- KBQA system supports 18 types of questions with accurate answers

## Architecture Onboarding

- **Component map**: Front-end (Vue, ECharts) <-> Back-end (Django) <-> Graph database (Neo4j) <-> Relational database (SQLite)
- **Critical path**: User uploads data → System constructs KG → User can query/view KG → KBQA system answers questions
- **Design tradeoffs**: Ease of use vs. flexibility in KG construction; automatic extraction vs. manual curation accuracy
- **Failure signatures**: Incomplete KG construction due to data format issues; incorrect question answering due to parsing errors; slow performance with large KGs
- **First 3 experiments**:
  1. Upload a small structured JSON file and verify KG construction
  2. Test audio preprocessing with sample medical dialogue
  3. Ask simple questions through KBQA system and verify answers
