# GeSubNet: Gene Interaction Inference for Disease Subtype Network Generation

## Quick Facts
- **arXiv ID**: 2410.13178
- **Source URL**: https://arxiv.org/abs/2410.13178
- **Reference count**: 29
- **Primary result**: Achieves 30.6%, 21.0%, 20.1%, and 56.6% average improvements across four graph evaluation metrics over four cancer datasets

## Executive Summary
GeSubNet addresses the challenge of creating disease-subtype specific gene interaction networks by integrating general knowledge databases with patient-specific gene expression profiles. The method employs a three-module framework that first learns distinct disease subtypes from gene expression data using VQ-VAE, then captures gene network representations from knowledge databases using Neo-GNN, and finally integrates these representations to generate subtype-specific networks. Biological validation through simulated gene knockout experiments demonstrates the method's ability to identify subtype-specific genes with high likelihood of impacting patient distribution shifts.

## Method Summary
GeSubNet is a multi-step representation learning framework that combines VQ-VAE and Neo-GNN architectures to generate subtype-specific gene interaction networks. The method first learns disease subtypes from patient gene expression profiles using a VQ-VAE module, then captures gene network representations from knowledge databases using a Neo-GNN module, and finally integrates these representations through an inference module that conditions graph generation on patient separation loss. The framework is trained in three independent phases with specific loss functions for each module, and evaluated across four cancer datasets using graph edit distance, similarity metrics, degree variation, and biological function enrichment.

## Key Results
- Outperforms traditional approaches with average improvements of 30.6%, 21.0%, 20.1%, and 56.6% across four graph evaluation metrics
- Successfully identifies subtype-specific genes with 83% likelihood of impacting patient distribution shifts in simulated knockout experiments
- Demonstrates effectiveness across four cancer datasets (BRCA, GBM, LGG, OV) using both KEGG and STRING knowledge databases

## Architecture Onboarding

- **Component map**: Patient-M (VQ-VAE) -> Graph-M (Neo-GNN) -> Infer-M (integration) -> Subtype-specific networks
- **Critical path**: Gene expression data → VQ-VAE (Patient-M) → Patient representations → Neo-GNN (Graph-M) → Gene network representations → Inference module → Subtype-specific networks
- **Design tradeoffs**: Discrete vs continuous latent representations (chosen VQ-VAE for better separation); Prior knowledge integration vs pure data-driven approach (chosen hybrid for biological relevance); Single-step vs multi-step learning (chosen multi-step for modularity)
- **Failure signatures**: Poor subtype separation in latent space (Patient-M issue); Generated networks too similar to prior knowledge (Graph-M not learning enough); Generated networks too sparse/disconnected (inference module balance issue)
- **First 3 experiments**:
  1. Test VQ-VAE module alone on gene expression data to verify it can separate known subtypes effectively
  2. Test Neo-GNN module alone on knowledge graph data to verify it can learn meaningful gene interactions
  3. Test integration of both modules on a small dataset to verify inference process generates meaningful subtype-specific networks
