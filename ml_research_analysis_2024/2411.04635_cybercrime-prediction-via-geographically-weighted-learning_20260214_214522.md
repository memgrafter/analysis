# Cybercrime Prediction via Geographically Weighted Learning

## Quick Facts
- arXiv ID: 2411.04635
- Source URL: https://arxiv.org/abs/2411.04635
- Reference count: 40
- Key outcome: GeogGNN achieved 69.6% accuracy with F1 score 0.6959, significantly outperforming standard NN (25.0%) and CNN (27.5%) baselines on synthetic 4-class cybercrime classification

## Executive Summary
This paper introduces GeogGNN, a graph neural network that incorporates geographical coordinates (latitude/longitude) into the adjacency matrix via a Gaussian kernel, enabling spatial variation to influence model predictions. The approach was tested on synthetic data centered in the Gulf Cooperation Council region for 4-class cybercrime classification. GeogGNN achieved 69.6% accuracy with F1 score 0.6959, significantly outperforming standard neural networks (25.0%) and convolutional neural networks (27.5%). The results demonstrate that explicitly modeling spatial relationships improves classification performance compared to treating coordinates as standard features. Additionally, a theoretical proof is provided showing that geographically weighted neural networks have lower classification error than standard neural networks when classes exhibit spatial continuity.

## Method Summary
The GeogGNN model constructs a geographically weighted adjacency matrix using a Gaussian kernel based on geographical distance between nodes. For nodes i and j with coordinates (x_i, y_i) and (x_j, y_j), the adjacency matrix entry is computed as A[i,j] = exp(-d[i,j]²/(2φ²)), where d[i,j] is the geographical distance. The model was compared against standard neural networks and CNNs that treat coordinates as regular features. The synthetic dataset consists of 4 classes with geographically distributed samples centered in the Gulf Cooperation Council region. Training was performed for 10,000-50,000 epochs with feature normalization and cross-entropy loss, evaluating using accuracy, F1 score, precision, recall, log-loss, AUC-ROC, and AUC-PR metrics.

## Key Results
- GeogGNN achieved 69.6% accuracy with F1 score 0.6959 on synthetic 4-class cybercrime classification
- Standard neural networks achieved 25.0% accuracy, and CNNs achieved 27.5% accuracy on the same task
- Theoretical proof demonstrates that geographically weighted neural networks have lower classification error than standard neural networks under spatial continuity assumptions

## Why This Works (Mechanism)
The GeogGNN approach works by incorporating spatial relationships directly into the model architecture through the adjacency matrix. By using a Gaussian kernel to weight connections between nodes based on their geographical distance, the model captures the spatial continuity inherent in many real-world phenomena, including cybercrime patterns. This allows the network to learn that geographically proximate instances are more likely to share similar characteristics and class labels. The adjacency matrix construction ensures that spatial information is not merely treated as another feature but is integrated into the fundamental structure of how information flows through the network during message passing operations.

Design tradeoffs: The key tradeoff is between model complexity and spatial accuracy. Using geographical coordinates in the adjacency matrix adds complexity but captures spatial relationships better than treating them as features. The Gaussian kernel parameter φ controls the spatial influence radius - larger values capture broader spatial patterns but may lose local specificity.

Failure signatures:
- Poor performance if treating coordinates as features rather than incorporating them into adjacency matrix
- Training instability with spiking loss function during early epochs
- Baseline models significantly outperforming GeogGNN suggests incorrect adjacency matrix construction or inappropriate kernel parameter selection
