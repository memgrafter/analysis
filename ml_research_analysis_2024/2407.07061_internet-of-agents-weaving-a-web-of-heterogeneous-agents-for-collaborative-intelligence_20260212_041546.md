# Internet of Agents: Weaving a Web of Heterogeneous Agents for Collaborative Intelligence

## Quick Facts
- arXiv ID: 2407.07061
- Source URL: https://arxiv.org/abs/2407.07061
- Reference count: 40
- Key outcome: Introduces IoA framework enabling collaboration among heterogeneous autonomous agents with superior performance on embodied AI and RAG benchmarks

## Executive Summary
This paper introduces IoA, a multi-agent framework designed to enable effective collaboration among heterogeneous autonomous agents inspired by the Internet's architecture. IoA addresses three key limitations of existing frameworks: lack of third-party agent integration, single-device simulation, and rigid communication pipelines. The framework introduces an agent integration protocol, instant-messaging-like architecture, and dynamic mechanisms for agent teaming and conversation flow control. Extensive experiments demonstrate that IoA consistently outperforms state-of-the-art baselines across various tasks, including embodied AI and RAG benchmarks, validating its ability to facilitate effective collaboration among diverse agents.

## Method Summary
The IoA framework consists of a server-client architecture where the server manages agent registration, discovery, and message routing, while clients act as wrappers for individual agents. The framework implements an agent integration protocol requiring agents to implement a simple `run(task_desc: str) -> str` interface for seamless integration. Communication is managed through a finite state machine inspired by Speech Act Theory, with states including discussion, synchronous task assignment, asynchronous task assignment, pause & trigger, and conclusion. Autonomous nested team formation allows agents to dynamically form hierarchical teams when sub-tasks require additional expertise. The framework was evaluated on 625 diverse tasks paired with 1500 dummy agent profiles, measuring team formation precision through Top@1, Top@10 recall rates, Mean Rank, and Mean Reciprocal Rank metrics.

## Key Results
- IoA consistently outperforms state-of-the-art baselines on embodied AI and RAG benchmarks
- Autonomous nested team formation achieves high precision in identifying relevant agents for tasks
- The instant-messaging-like architecture enables seamless integration of third-party agents

## Architecture Onboarding

- Component map: Server (central hub for agent registration, discovery, and message routing) -> Client (wrappers for individual agents providing communication functionalities)

- Critical path: Task assignment → Team formation → Message exchange → Task execution → Result aggregation

- Design tradeoffs:
  - Flexibility vs. complexity: IoA aims to provide a flexible platform for diverse agents but may introduce complexity in managing the interactions
  - Centralized vs. distributed: IoA relies on a central server for coordination but aims to support distributed agent collaboration

- Failure signatures:
  - Agents failing to integrate or communicate properly
  - Inefficient team formation or task allocation
  - Communication breakdowns or misunderstandings among agents

- First 3 experiments:
  1. Integrate a simple agent (e.g., a basic chatbot) and verify its ability to communicate and collaborate with other agents in IoA
  2. Test the team formation mechanism by assigning a simple task and observing how agents autonomously form teams to address the task
  3. Evaluate the conversation flow control by monitoring the state transitions and speaker selections during a multi-turn dialogue among agents

- Basis in paper: [explicit] "To evaluate the economic feasibility and potential for optimization of the IoA, we conduct a cost analysis on the open-ended instruction benchmark"
- Why unresolved: The cost analysis only examined a specific case with two third-party agents
- What evidence would resolve it: Comprehensive cost modeling across varying numbers of agents (e.g., 2, 5, 10, 20 agents) and agent types, measuring both computational costs and performance gains

### Open Question 3
- Question: What specific improvements in LLM alignment would most effectively reduce suboptimal communication patterns in IoA?
- Basis in paper: [explicit] "we observed unexpected and suboptimal communication patterns that contributed to the high communication cost... This insight raises new research questions regarding the agent alignment of LLMs"
- Why unresolved: The paper identifies the problem but doesn't specify which alignment techniques would be most effective for improving agent communication
- What evidence would resolve it: Comparative experiments testing different alignment approaches (e.g., specialized fine-tuning, reward modeling for communication efficiency, instruction tuning for dialogue management) and measuring their impact on communication quality and cost

## Limitations
- Limited empirical validation scope across diverse real-world scenarios
- Agent integration protocol may not accommodate complex agent architectures requiring sophisticated interaction patterns
- Reliance on LLMs for conversation flow control introduces variability and potential suboptimal decision-making

## Confidence
- Agent Integration Protocol: Medium confidence - clear conceptual framework but lacks edge case discussion
- Autonomous Conversation Flow Control: Medium confidence - theoretically sound but LLM dependency introduces variability
- Autonomous Nested Team Formation: Medium-High confidence - clear hierarchical structure but scalability concerns unexplored
