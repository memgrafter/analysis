# An Independent Implementation of Quantum Machine Learning Algorithms in Qiskit for Genomic Data

## Quick Facts
- arXiv ID: 2405.09781
- Source URL: https://arxiv.org/abs/2405.09781
- Authors: Navneet Singh; Shiva Raj Pokhrel
- Reference count: 7
- Primary result: Quantum ML algorithms (QSVC, Pegasos-QSVC, VQC, QNN) with quantum feature maps achieve superior genomic sequence classification performance over baseline implementations

## Executive Summary
This paper presents an independent implementation of four quantum machine learning algorithms‚ÄîQuantum Support Vector Classifier (QSVC), Pegasos-QSVC, Variational Quantum Classifier (VQC), and Quantum Neural Network (QNN)‚Äîusing Qiskit for genomic sequence classification. The authors evaluate three feature mapping techniques (ZFeatureMap, ZZFeatureMap, and PauliFeatureMap) to encode genomic data into quantum states and compare their performance. Using a dataset of 100,000 genomic sequences (democoding vs. intergenomic), the implementation demonstrates substantial improvements in classification metrics including accuracy, precision, recall, F1-Score, and AUROC, with enhanced convergence in objective functions compared to baseline approaches.

## Method Summary
The study implements quantum ML algorithms in Qiskit for genomic sequence classification using a dataset of 100,000 sequences. The pipeline involves text vectorization of genomic sequences, PCA dimensionality reduction, and encoding using three quantum feature mapping techniques (ZFeatureMap, ZZFeatureMap, PauliFeatureMap) with 4 qubits. Four algorithms are implemented: QSVC using quantum kernels, Pegasos-QSVC with stochastic optimization, VQC with parameterized quantum circuits, and QNN with quantum neural network layers. Classical optimizers are used for training, and performance is evaluated using standard classification metrics including accuracy, precision, recall, F1-Score, and AUROC.

## Key Results
- Quantum feature maps (ZFeatureMap, ZZFeatureMap, PauliFeatureMap) enable substantial improvements in genomic sequence classification performance
- Enhanced convergence in objective functions observed for QNN and VQC implementations compared to baseline approaches
- Superior classification metrics (accuracy, precision, recall, F1-Score, AUROC) achieved across all evaluated algorithms
- Quantum-enhanced feature spaces demonstrate significantly better performance than classical implementations for genomic data classification

## Open Questions the Paper Calls Out

### Open Question 1
- Question: How do the proposed quantum machine learning algorithms scale with larger genomic datasets beyond the 100,000 sequences tested?
- Basis in paper: [explicit] The authors state "evaluations with larger datasets will further illuminate the potential of these advancements" and mention their preliminary analysis demonstrates improved performance on the 100,000 sequence subset.
- Why unresolved: The current implementation only tests on 100,000 sequences, which is relatively small for genomic data applications. The authors explicitly acknowledge the need for evaluation on larger datasets to validate scalability.
- What evidence would resolve it: Performance metrics (accuracy, precision, recall, F1-score, AUROC) and computational resource requirements when tested on datasets of varying sizes (e.g., 1 million, 10 million, 100 million sequences).

### Open Question 2
- Question: What is the impact of different quantum hardware architectures and noise levels on the performance of the implemented algorithms?
- Basis in paper: [inferred] The authors use Qiskit for implementation and mention ongoing detailed investigations, but do not address hardware-specific variations or noise mitigation strategies.
- Why unresolved: The implementation is described as software-based using Qiskit, with no discussion of how different quantum processors or noise levels might affect results, despite this being a critical consideration for practical quantum computing applications.
- What evidence would resolve it: Comparative performance analysis across different quantum hardware platforms (superconducting, trapped ion, photonic) with varying noise levels and error rates.

### Open Question 3
- Question: How do the quantum feature mapping techniques compare to classical feature extraction methods for genomic sequence classification in terms of both performance and computational efficiency?
- Basis in paper: [explicit] The authors evaluate three feature mapping techniques (ZFeatureMap, ZZFeatureMap, PauliFeatureMap) but do not compare them to classical feature extraction methods commonly used in genomic analysis.
- Why unresolved: While the paper focuses on quantum feature mapping techniques, it does not benchmark against classical approaches like k-mer counting, word embeddings, or other established genomic feature extraction methods.
- What evidence would resolve it: Direct comparison of classification performance and computational time between quantum feature mapping and classical feature extraction methods on the same genomic datasets.
