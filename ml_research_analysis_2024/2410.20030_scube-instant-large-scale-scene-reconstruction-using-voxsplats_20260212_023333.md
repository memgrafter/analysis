# SCube: Instant Large-Scale Scene Reconstruction using VoxSplats

## Quick Facts
- arXiv ID: 2410.20030
- Source URL: https://arxiv.org/abs/2410.20030
- Reference count: 40
- Primary result: Novel large-scale scene reconstruction using VoxSplat hybrid representation combining 3D Gaussians with sparse voxel scaffold

## Executive Summary
SCube introduces a novel method for large-scale 3D scene reconstruction from sparse images using the VoxSplat representation. This hybrid approach combines 3D Gaussians with a sparse voxel scaffold to enable rapid reconstruction of millions of Gaussians spanning hundreds of meters. The two-stage pipeline uses a voxel latent diffusion model to generate high-resolution sparse voxel grids from input images, followed by a feedforward network that predicts Gaussian splats within each voxel. SCube achieves impressive novel view synthesis performance on the Waymo dataset while reconstructing scenes in just 20 seconds from as few as 3 non-overlapping input images.

## Method Summary
SCube employs a two-stage approach for large-scale scene reconstruction. First, a voxel latent diffusion model processes input images to generate a high-resolution sparse voxel grid representing the scene geometry. Second, a feedforward network predicts Gaussian splats within each voxel, capturing both geometric and appearance information. The VoxSplat representation serves as a hybrid between explicit voxel structures and implicit Gaussian splats, enabling efficient storage and rendering while maintaining high visual fidelity. This design allows SCube to handle millions of Gaussians with a 1024³ voxel grid, achieving rapid reconstruction times suitable for instant scene capture.

## Key Results
- Achieves 25.90 PSNR, 0.77 SSIM, and 0.45 LPIPS for novel view synthesis on Waymo dataset
- Reconstructs millions of Gaussians with 1024³ voxel grid spanning hundreds of meters in 20 seconds
- Outperforms baselines using as few as 3 non-overlapping input images
- Demonstrates applications in LiDAR simulation and text-to-scene generation

## Why This Works (Mechanism)
The VoxSplat representation combines the spatial efficiency of sparse voxel grids with the visual fidelity of Gaussian splats. The voxel latent diffusion model effectively captures global scene structure from sparse inputs, while the feedforward network refines local details by predicting Gaussians within each voxel. This hybrid approach balances computational efficiency with reconstruction quality, enabling instant large-scale scene reconstruction that would be infeasible with purely explicit or implicit representations.

## Architecture Onboarding

**Component Map**: Input Images -> Voxel Latent Diffusion -> Sparse Voxel Grid -> Feedforward Network -> Gaussian Splats -> Rendered Scene

**Critical Path**: The voxel latent diffusion model forms the bottleneck, as it must generate the initial scene structure before Gaussian splat prediction can occur. The feedforward network's performance is critical for final visual quality.

**Design Tradeoffs**: The voxel resolution (1024³) versus Gaussian splat density represents a key tradeoff between geometric detail and appearance fidelity. Higher voxel resolution enables finer geometric structure but increases memory requirements.

**Failure Signatures**: Poor input image quality or extreme sparsity can lead to incomplete voxel grids, causing artifacts in the Gaussian splat prediction stage. Limited input views may result in missing scene regions.

## Limitations

- Evaluation primarily limited to Waymo dataset, potentially limiting generalizability to other domains
- Performance with extremely sparse inputs (3 images) may not translate to more challenging scenarios
- Hardware configuration details for the 20-second reconstruction time are not fully specified
