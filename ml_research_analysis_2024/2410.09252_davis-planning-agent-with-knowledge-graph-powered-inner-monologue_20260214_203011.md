# DAVIS: Planning Agent with Knowledge Graph-Powered Innerologue

## Quick Facts
- arXiv ID: 2410.09252
- Source URL: https://arxiv.org/abs/2410.09252

- Primary result: Achieves 65.06/100 average score on ScienceWorld benchmark, outperforming previous approaches by >1.8x

## Executive Summary
ReasonPlanner is an autonomous planning agent that addresses interactive reasoning challenges in scientific environments through a novel combination of Temporal Knowledge Graph world modeling and LLM-powered trajectory generation. The agent generates hypothetical trajectories in a knowledge-augmented space before execution, enabling interpretable and resource-efficient planning. On the ScienceWorld benchmark of 30 elementary science tasks, ReasonPlanner achieves an average score of 65.06 out of 100, representing more than 1.8x improvement over previous approaches while requiring only 2GB GPU memory and no gradient updates.

## Method Summary
The approach leverages a Temporal Knowledge Graph as a World Model to enable model-based planning, combined with an actor-critic execution framework. The system uses LLMs to generate trajectories in a hypothetical space before execution, with the ability to query its knowledge base through a k-hop question-answering system. This enables the agent to reason about potential actions and their consequences in a structured, interpretable manner while maintaining computational efficiency.

## Key Results

## Why This Works (Mechanism)
The Temporal Knowledge Graph serves as a structured world model that captures relationships and temporal dynamics in scientific environments. By generating trajectories in this hypothetical space before execution, the agent can reason about potential outcomes and avoid costly mistakes. The k-hop question-answering system enables efficient querying of relevant knowledge, while the actor-critic framework provides a balanced approach to exploration and exploitation during execution.

## Architecture Onboarding

Component Map: Temporal Knowledge Graph -> LLM Trajectory Generator -> k-hop QA System -> Actor-Critic Executor

Critical Path: Knowledge Graph construction → Trajectory generation → Knowledge query → Action execution → Performance evaluation

Design Tradeoffs:
- Static knowledge graph vs. dynamic learning: Prioritizes interpretability and efficiency over adaptability
- LLM-based planning vs. traditional search: Leverages language understanding at potential cost of precision
- No gradient updates: Enables deployment without continuous training infrastructure

Failure Signatures:
- Knowledge graph incompleteness leading to poor planning decisions
- LLM hallucinations affecting trajectory generation quality
- Suboptimal actor-critic balance causing inefficient exploration
- k-hop limitations preventing access to critical knowledge
