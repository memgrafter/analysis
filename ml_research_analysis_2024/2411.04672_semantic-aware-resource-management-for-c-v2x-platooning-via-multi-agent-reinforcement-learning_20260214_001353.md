# Semantic-Aware Resource Management for C-V2X Platooning via Multi-Agent Reinforcement Learning

## Quick Facts
- arXiv ID: 2411.04672
- Source URL: https://arxiv.org/abs/2411.04672
- Reference count: 40
- Key outcome: This paper introduces a semantic-aware resource management framework for cellular vehicle-to-everything (C-V2X) platooning systems using multi-agent reinforcement learning (MARL). The framework addresses the challenge of efficiently managing communication resources in dynamic vehicular environments by incorporating semantic communication, which transmits extracted features of information rather than raw data.

## Executive Summary
This paper presents SAMRAMARL, a novel framework for resource management in C-V2X platooning systems that leverages semantic communication and multi-agent reinforcement learning. The approach dynamically allocates channels, power, and semantic symbol lengths based on contextual importance of transmitted information, optimizing both quality of experience (QoE) and success rate of semantic information transmission (SRS). The distributed MARL architecture enables scalable decision-making across multiple platoons while reducing signaling overhead compared to centralized methods.

## Method Summary
The SAMRAMARL framework constructs mathematical models for semantic communication using DeepSC (text) and MU-DeepSC (multi-modal) models, then formulates an optimization problem to maximize QoE and SRS metrics. The algorithm employs a multi-agent actor-critic approach where each platoon leader acts as an independent agent making decisions based on local observations. The framework uses twin global critics and local critics to guide cooperative behavior while maintaining scalability. The method is trained on European Parliament dataset for text and CLEVR dataset for multi-modal data, with communication settings based on 3GPP TR 36.885.

## Key Results
- SAMRAMARL achieves significant gains in quality of experience (QoE) compared to baseline methods
- The framework improves success rate of semantic information transmission (SRS) in vehicle-to-vehicle links
- Communication delay is reduced compared to traditional resource allocation approaches

## Why This Works (Mechanism)

### Mechanism 1
- Claim: The SAMRAMARL algorithm achieves superior QoE and SRS by dynamically adjusting semantic symbol lengths based on contextual importance.
- Mechanism: The algorithm uses MARL to independently optimize channel assignment, power allocation, and semantic symbol length for each platoon. By incorporating semantic communication models (DeepSC for text, MU-DeepSC for multi-modal), the system extracts meaningful features rather than raw data, reducing redundancy. The contextual importance weighting ensures that high-priority messages are allocated more resources, improving both QoE and SRS metrics.
- Core assumption: The semantic communication models accurately capture task-relevant features, and the MARL agents can effectively learn optimal resource allocation policies in dynamic environments.
- Evidence anchors:
  - [abstract] "The algorithm can dynamically allocate channels and power and determine semantic symbol length based on the contextual importance of the transmitted information"
  - [section III-A] "The system supports two communication modes: single-modal tasks and multi-modal tasks. For single-modal communication, the DeepSC model is employed... For multi-modal tasks, the MU-DeepSC model is utilized"
  - [corpus] "Semantic-Aware Spectrum Sharing in Internet of Vehicles Based on Deep Reinforcement Learning" - supports the concept of semantic communication in vehicular networks
- Break condition: If the semantic models fail to accurately extract meaningful features or if the MARL agents cannot adapt to rapidly changing channel conditions, performance will degrade significantly.

### Mechanism 2
- Claim: The QoE metric formulation balances semantic accuracy and transmission rate, enabling vehicles to prioritize based on application needs.
- Mechanism: QoE is defined as a weighted sum of semantic rate and semantic accuracy, with adjustable weights allowing different vehicles to prioritize differently. The sigmoid functions ensure smooth gradients for optimization. By explicitly managing the trade-off between rate and accuracy, the system can optimize for both safety-critical messages (high accuracy) and general information (higher rate).
- Core assumption: The semantic similarity and rate metrics accurately reflect user experience and can be optimized through the MARL framework.
- Evidence anchors:
  - [abstract] "We propose the quality of experience (QoE) metric based on semantic similarity and semantic rate"
  - [section II-B] "The QoE for vehicle m in platoon n is defined as: QoEnm = Σm∈Mn [ωmScoreR(ϕm) + (1 − ωm)ScoreA(ξm)]"

- Break condition: If the sigmoid scoring functions do not accurately capture user preferences or if the weights cannot be properly tuned, the QoE optimization may not reflect actual user experience.

### Mechanism 3
- Claim: The distributed MARL approach reduces signaling overhead and improves scalability compared to centralized methods.
- Mechanism: Each platoon leader acts as an independent agent making decisions based on local observations (channel state, interference, load). The shared replay buffer enables knowledge transfer between agents without requiring global coordination. The global critic network guides overall system performance while local critics optimize individual agent behavior. This architecture scales to large numbers of platoons without centralized bottlenecks.
- Core assumption: Local observations provide sufficient information for effective decision-making and that the global reward signal can guide cooperative behavior without explicit coordination.
- Evidence anchors:
  - [abstract] "To address these challenges, we propose a distributed SAMRA framework leveraging multi-agent reinforcement learning (MARL), termed SAMRAMARL"
  - [section III] "Each PL interacts with the environment as an agent and makes decisions through the corresponding strategies"
  - [corpus] "Graph Neural Networks and Deep Reinforcement Learning Based Resource Allocation for V2X Communications" - supports distributed learning approaches in vehicular networks
- Break condition: If local observations become too limited due to network partitioning or if the global reward signal is too weak to coordinate behavior, the distributed approach may fail to achieve optimal performance.

## Architecture Onboarding

- Component map:
  - Vehicle sensors -> Semantic encoder -> Agent decision -> Channel/power allocation -> OFDM subchannel -> Wireless channel with interference -> Receiver -> Semantic decoder -> QoE/SRS calculation

- Critical path:
  1. Vehicle detects event and generates semantic message
  2. Semantic encoder extracts features and maps to symbols
  3. Agent selects resource allocation (channel, power, symbol length)
  4. Message transmitted over wireless channel with interference
  5. Receiver decodes semantic symbols back to message
  6. QoE and SRS metrics calculated and fed back to agents

- Design tradeoffs:
  - Centralized vs. Distributed: Centralized offers global optimization but high overhead; distributed scales better but may have coordination challenges
  - Semantic vs. Traditional: Semantic reduces redundancy but requires accurate feature extraction; traditional is simpler but less efficient
  - Single-modal vs. Multi-modal: Multi-modal provides richer information but increases complexity; single-modal is simpler but less informative

- Failure signatures:
  - Poor convergence: Rewards fluctuate or plateau at suboptimal levels
  - High signaling overhead: Agents require frequent coordination
  - Inaccurate semantic extraction: Semantic similarity metrics show poor performance
  - Resource starvation: Some vehicles consistently receive poor channel allocations

- First 3 experiments:
  1. Single platoon, fixed channel conditions, compare SAMRAMARL vs. DDPG with and without semantic communication
  2. Multiple platoons, varying channel conditions, measure convergence speed and stability
  3. Stress test with high semantic demand, measure QoE and SRS under load conditions

## Limitations
- The framework relies heavily on accurate semantic feature extraction, which may not generalize well to all communication types beyond text and structured multi-modal data
- The distributed MARL approach assumes local observations provide sufficient information for decision-making, which may not hold in highly dynamic or congested network conditions
- Scalability to very large networks with hundreds of platoons remains unverified, and the framework's robustness to extreme channel conditions or malicious attacks is not addressed
