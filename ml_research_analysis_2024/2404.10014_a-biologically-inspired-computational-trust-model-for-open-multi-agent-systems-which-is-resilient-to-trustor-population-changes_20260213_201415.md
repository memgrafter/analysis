# A biologically inspired computational trust model for open multi-agent systems which is resilient to trustor population changes

## Quick Facts
- arXiv ID: 2404.10014
- Source URL: https://arxiv.org/abs/2404.10014
- Reference count: 12
- Primary result: FIRE model excels at handling trustee population changes while CA model shows resilience to trustor population changes in dynamic multi-agent systems

## Executive Summary
This paper compares two computational trust models, CA (Create Assemblies) and FIRE, under dynamic conditions in open multi-agent systems. The CA model, inspired by synaptic plasticity in the human brain, operates from the trustee's perspective where agents decide whether to perform tasks based on their own trust values. FIRE is a well-established model that aggregates trust information from multiple sources including interaction trust, witness reputation, role-based trust, and certified reputation. The study evaluates performance under conditions of continuous changes in trustee and trustor populations, as well as changes in trustees' abilities to perform tasks.

The results show that FIRE is superior in handling changes in the trustee population, while CA is more resilient to changes in the trustor population. When trustees switch performance profiles, FIRE clearly outperforms CA, although both models' performances are significantly impacted by this environmental change. The paper concludes that learning to use the appropriate trust model based on the dynamic conditions in effect could maximize the trustor's benefits.

## Method Summary
The study uses a simulation testbed with spherical world topology containing 100 providers (10 good, 40 ordinary, 5 intermittent, 45 bad) and 500 consumers. Three consumer groups are compared: NoTrustModel, FIRE, and CA. Agents move randomly within the world and service requests when needed. The simulation runs for 5000 time units with 30 independent simulation runs (NISR=30) to establish statistical significance. Two-sample t-tests with 95% confidence level are used to compare average utility gain (UG) per interaction across different experimental conditions including static environments, provider population changes (2%, 5%, 10%), consumer population changes, and dynamic trustee profiles.

## Key Results
- FIRE outperforms CA when handling changes in the trustee population due to its aggregation of multiple trust information sources
- CA shows resilience to changes in the trustor population because it does not rely on witness reputation information
- Both models experience significant performance degradation when trustees switch performance profiles, with FIRE maintaining a clear advantage
- Neither model consistently outperforms the other across all dynamic conditions, suggesting the need for adaptive model selection

## Why This Works (Mechanism)

### Mechanism 1
- Claim: CA model is resilient to trustor population changes because it does not rely on witness reputation information.
- Mechanism: In CA, the trustee decides whether to perform a task based on its own trust value stored in connection weights. This eliminates the need for trustors to collect and share opinions about providers, making the model independent of trustor population dynamics.
- Core assumption: The trustee's own trust value accurately reflects its capability to perform tasks.
- Evidence anchors:
  - [abstract] "Importantly, in CA approach, agents do not exchange information which naturally creates the expectation that CA model is immune to various kinds of disinformation"
  - [section 2.3] "FIRE, unlike CA, is dependent on the consumer population for Witness Reputation"

### Mechanism 2
- Claim: FIRE is more resilient to trustee population changes because it aggregates trust information from multiple sources including witness reputation.
- Mechanism: FIRE uses Interaction Trust, Witness Reputation, Role-based Trust, and Certified Reputation modules. When trustee population changes occur, the model can still function using the remaining trust information sources, particularly the more stable certified reputation data.
- Core assumption: Multiple independent trust information sources provide redundancy when one source becomes unreliable.
- Evidence anchors:
  - [section 2.2] "FIRE [5] is a well-known trust and reputation model that follows the distributed, decentralized approach. It incorporates the following four modules"
  - [section 5] "When witnesses leave the system, they take with them their opinions and Witness Reputation module cannot work"

### Mechanism 3
- Claim: CA's performance degrades with frequent trustee population changes because newcomer trustees must learn their capabilities from scratch.
- Mechanism: CA relies on trustees maintaining connection weights that represent their trust values. When new trustees enter the system, they start with default weights and must accumulate experience to establish accurate trust values, leading to initial poor performance.
- Core assumption: Trustee capabilities can be accurately learned through experience and reflected in connection weights.
- Evidence anchors:
  - [section 5] "CA model is based on the providers' knowledge of their capabilities, which is stored in the form of weights of the connections they create"
  - [section 4.2] "CA's performance is significantly hampered by bad newcomers who provide bad quality services, as they are unfamiliar with their capabilities and must learn from scratch"

## Architecture Onboarding

- Component map: Testbed environment with spherical world topology -> Provider agents with four performance profiles -> Consumer agents divided into three groups -> Communication layer for message broadcasting -> Trust evaluation modules for FIRE -> Connection weight management system for CA -> Performance monitoring and utility calculation system

- Critical path: Consumer detects task need and broadcasts request -> Providers receive request and establish/receive connection weights -> Providers evaluate their trust value against threshold -> Providers decide whether to undertake task -> Task execution and result reporting -> Trust value update based on outcome -> Utility calculation for performance evaluation

- Design tradeoffs:
  - CA: No witness reputation overhead but slower learning for new providers
  - FIRE: Faster learning through shared information but vulnerable to trustor population changes
  - Privacy: CA preserves privacy by eliminating information exchange, FIRE requires sharing ratings
  - Scalability: CA scales better with trustor population size, FIRE scales better with trustee population size

- Failure signatures:
  - CA: Consistently low performance after trustee population changes indicates inability to learn new provider capabilities
  - FIRE: Performance degradation with trustor population changes indicates insufficient witness information
  - Both: Sudden performance drops when providers switch profiles indicate model inability to track dynamic capabilities

- First 3 experiments:
  1. Static environment comparison: Compare CA, FIRE, and NoTrustModel performance without any population changes to establish baseline behavior
  2. Trustee population change: Gradually increase provider replacement rate from 2% to 10% to identify resilience thresholds
  3. Trustor population change: Gradually increase consumer replacement rate from 2% to 10% to test CA's resilience advantage

## Limitations

- The experimental results are based on simulations with fixed agent populations and specific performance profiles that may not generalize to real-world scenarios
- The study focuses primarily on utility metrics without considering computational overhead, privacy implications, or security vulnerabilities that might affect real-world deployment
- The paper does not explore intermediate hybrid approaches that could potentially combine the strengths of both CA and FIRE models

## Confidence

- **High Confidence**: FIRE's superior performance under trustee population changes is well-supported by the experimental data and aligns with the theoretical advantage of aggregating multiple trust information sources
- **Medium Confidence**: CA's resilience to trustor population changes is demonstrated, but the magnitude of this advantage could vary with different simulation parameters or real-world conditions
- **Low Confidence**: The claim that learning to use the appropriate trust model based on dynamic conditions could maximize benefits is speculative and not directly tested in the experiments
