# Compact 3D Gaussian Splatting For Dense Visual SLAM

## Quick Facts
- arXiv ID: 2403.11247
- Source URL: https://arxiv.org/abs/2403.11247
- Authors: Tianchen Deng; Yaohui Chen; Leyan Zhang; Jianfei Yang; Shenghai Yuan; Jiuming Liu; Danwei Wang; Hesheng Wang; Weidong Chen
- Reference count: 40
- One-line primary result: Achieves 226% increase in rendering speed and over 2.21× compression in memory usage while maintaining state-of-the-art image reconstruction quality

## Executive Summary
This paper addresses the high memory and storage costs in 3D Gaussian-based SLAM systems by proposing a compact 3D Gaussian Splatting SLAM framework. The core innovation involves three key components: a sliding window-based masking strategy to remove redundant Gaussian ellipsoids, a geometry codebook-based quantization method to compress geometric attributes, and a local-to-global bundle adjustment method with reprojection loss for robust pose estimation. The proposed method significantly improves computational efficiency while maintaining high-quality scene reconstruction.

## Method Summary
The method involves a sliding window-based masking strategy to remove redundant Gaussian ellipsoids during SLAM operation, followed by a geometry codebook-based quantization method that uses residual vector quantization (R-VQ) to compress scale and rotation attributes. A local-to-global bundle adjustment method with reprojection loss is implemented for accurate pose estimation. The system processes sequential RGB-D frames with known camera intrinsics, maintaining a global keyframe database and performing iterative optimization of both scene representation and camera poses.

## Key Results
- 226% increase in rendering speed compared to baseline SplaTAM system
- Over 2.21× compression in memory usage while maintaining reconstruction quality
- Achieves state-of-the-art image reconstruction quality with PSNR, SSIM, and LPIPS metrics comparable to original SplaTAM

## Why This Works (Mechanism)

### Mechanism 1
- Claim: The sliding window-based masking strategy effectively reduces redundant Gaussian ellipsoids without degrading reconstruction quality.
- Mechanism: The mask operates only on Gaussians within the current viewing frustum and uses a sliding window reset strategy to prevent continuous optimization that would eliminate all Gaussians. By combining volume and opacity in the mask calculation, it selectively removes unnecessary Gaussians while preserving essential geometry.
- Core assumption: Redundant Gaussians can be identified and removed based on their visibility and hit frequency within a local temporal window without affecting global scene consistency.
- Evidence anchors:
  - [abstract]: "A sliding window-based masking strategy is first proposed to reduce the redundant ellipsoids"
  - [section 3.2]: "We utilize Ŝn, ôn in the color and depth rendering" and "When moving to the bound of the sliding window, we remove the redundant Gaussian ellipsoids by evaluating their hit frequency in conjunction with the learned mask"

- Break condition: If the local window is too small, important geometry might be prematurely removed; if too large, memory benefits diminish and optimization becomes inefficient.

### Mechanism 2
- Claim: The geometry codebook-based quantization method significantly compresses geometric attributes while maintaining reconstruction quality.
- Mechanism: The residual vector quantization (R-VQ) method cascades multiple stages of vector quantization, storing only codebook indices rather than full geometric parameters. This exploits the high similarity in Gaussian ellipsoid geometry across scenes.
- Core assumption: The geometric attributes (scale and rotation) of Gaussian ellipsoids exhibit sufficient similarity across the scene to allow effective compression through vector quantization.
- Evidence anchors:
  - [abstract]: "Then we observe that the covariance matrix (geometry) of most 3D Gaussian ellipsoids are extremely similar, which motivates a novel geometry codebook to compress 3D Gaussian geometric attributes"
  - [section 3.3]: "We incorporate the residual vector quantization (R-VQ) to compress the scale and rotation" and "After this, we can only store the codebook compressed scale and rotation vector, which can significantly reduce storage and memory usage"
  - [corpus]: Weak evidence - related papers mention compression techniques but don't specifically address R-VQ for geometric attributes in SLAM
- Break condition: If scene geometry lacks sufficient similarity, quantization error increases and reconstruction quality degrades.

### Mechanism 3
- Claim: The local-to-global bundle adjustment with reprojection loss improves camera tracking accuracy and eliminates cumulative errors.
- Mechanism: The method maintains a global keyframe database and performs bundle adjustment using all historical observations, incorporating reprojection loss with SIFT features to improve geometric consistency.
- Core assumption: Incorporating reprojection error from 2D features into the optimization loop provides additional constraints that improve pose estimation beyond what's available from Gaussian splatting alone.
- Evidence anchors:
  - [abstract]: "Robust and accurate pose estimation is achieved by a global bundle adjustment method with reprojection loss"
  - [section 3.4]: "We introduce this loss for the first time to improve the scene's geometric representation and consistency further" and "Our local-to-global BA method can effectively reduce cumulative errors and enhance the robustness of pose estimation"
  - [corpus]: Weak evidence - related papers mention bundle adjustment but don't specifically address local-to-global BA with reprojection loss for Gaussian splatting SLAM
- Break condition: If keyframe database becomes too large, computational cost may outweigh benefits; if reprojection error dominates, it may introduce bias from feature detection errors.

## Architecture Onboarding

- Component map: RGB-D input -> Gaussian ellipsoid creation -> sliding window masking -> geometry codebook quantization -> rendering -> reprojection loss calculation -> bundle adjustment -> pose update

- Critical path: The most critical sequence is: RGB-D input → Gaussian ellipsoid creation → sliding window masking → geometry codebook quantization → rendering → reprojection loss calculation → bundle adjustment → pose update

- Design tradeoffs:
  - Memory vs. quality: More aggressive masking and quantization reduce memory but may impact reconstruction quality
  - Speed vs. accuracy: Faster iterations may converge to suboptimal solutions
  - Local vs. global optimization: Local window masking is efficient but requires global BA for accuracy

- Failure signatures:
  - Excessive Gaussian removal: Reconstruction quality degrades, holes appear in rendered images
  - Over-compression: Geometric distortions become visible, especially in high-frequency details
  - Cumulative drift: Without proper global BA, trajectory errors accumulate over long sequences

- First 3 experiments:
  1. Baseline comparison: Run the original SplaTAM system and measure memory usage, rendering speed, and reconstruction quality
  2. Masking ablation: Implement the sliding window mask but disable it initially, then gradually enable to measure impact on Gaussian count and performance
  3. Codebook compression: Implement the geometry codebook with different codebook sizes to find the optimal compression ratio that maintains quality
