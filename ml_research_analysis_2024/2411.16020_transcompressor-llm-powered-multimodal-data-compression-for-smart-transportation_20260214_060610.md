# TransCompressor: LLM-Powered Multimodal Data Compression for Smart Transportation

## Quick Facts
- arXiv ID: 2411.16020
- Source URL: https://arxiv.org/abs/2411.16020
- Reference count: 32
- Key outcome: First framework using LLMs for multimodal sensor data compression in smart transportation achieves high reconstruction accuracy without fine-tuning

## Executive Summary
This paper introduces TransCompressor, a novel framework that leverages Large Language Models (LLMs) for multimodal sensor data compression in smart transportation systems. The system employs skip sampling and data rescaling to compress transportation sensor data (barometer, speed, and altitude) from buses, taxis, and MTRs, then reconstructs it using GPT-4 through well-crafted prompts. Evaluations across different transportation modes and compression ratios show that TransCompressor achieves high reconstruction accuracy without requiring fine-tuning or domain-specific training, establishing a foundation for integrating LLMs into smart transportation systems for enhanced data management and operational efficiency.

## Method Summary
TransCompressor processes transportation sensor data through skip sampling to reduce data transmission volume while preserving temporal patterns, followed by data rescaling to normalize values to [0,1] range with two decimal precision. The compressed data is then fed to GPT-4 for zero-shot reconstruction using carefully crafted prompts. The framework handles multimodal sensor data including barometer, speed, and altitude measurements from various transportation modes (buses, taxis, and MTRs), achieving data compression while maintaining reconstruction accuracy through the LLM's interpolation capabilities.

## Key Results
- Achieves high reconstruction accuracy across different compression ratios without requiring fine-tuning
- Reconstruction accuracy improves with higher compression ratios as more original data is retained
- Demonstrates effective zero-shot reconstruction capability of LLMs for transportation sensor data

### Mechanism 2
- LLMs can serve as zero-shot reconstructors using their knowledge base without domain-specific training
- The LLM leverages its general knowledge base and interpolation capabilities to fill in missing sensor values based on surrounding data patterns
- Core assumption: LLMs possess sufficient general knowledge about sensor data patterns and interpolation techniques to reconstruct missing values accurately
- Evidence anchors: Experimental results demonstrate LLMs effectively execute zero-shot reconstruction with compressed transportation sensor data
- Break condition: If sensor data patterns fall outside the LLM's knowledge base or are too complex for general interpolation knowledge

### Mechanism 3
- Data rescaling to [0,1] range with 2 decimal precision optimizes LLM processing efficiency
- Rescaling normalizes data values to a standard range and truncation reduces token length, improving LLM processing speed and accuracy
- Core assumption: LLMs process normalized numerical data more efficiently and truncated values maintain sufficient precision for reconstruction
- Evidence anchors: Goal of rescaling is to prepare data for processing by ensuring conformity to standard scale; truncated data is then ready to be fed into LLMs
- Break condition: If truncation removes critical precision needed for accurate reconstruction or normalization introduces significant distortion
