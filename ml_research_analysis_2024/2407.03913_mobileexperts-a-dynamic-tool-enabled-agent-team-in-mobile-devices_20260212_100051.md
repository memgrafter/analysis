# MobileExperts: A Dynamic Tool-Enabled Agent Team in Mobile Devices

## Quick Facts
- **arXiv ID**: 2407.03913
- **Source URL**: https://arxiv.org/abs/2407.03913
- **Reference count**: 35
- **Primary result**: 22% reduction in reasoning costs through dynamic multi-agent team assembly

## Executive Summary
MobileExperts introduces a multimodal multi-agent framework for mobile device operation that addresses the challenge of complex task handling and high reasoning costs in existing mobile agents. The system dynamically assembles teams of specialized agents based on task requirements, with each agent independently formulating tools and memories through exploration. A dual-layer planning mechanism coordinates collaboration between team-level task distribution and expert-level atomic task decomposition. The framework introduces code-combination-based tool formulation and a proactive working memory system.

## Method Summary
MobileExperts implements a multimodal multi-agent framework that dynamically assembles specialized agent teams for mobile device tasks. The system features a dual-layer planning mechanism where team-level planning distributes complex tasks across multiple experts, while expert-level planning decomposes tasks into atomic operations. Each agent independently formulates required tools through code-combination approaches and maintains proactive working memory for long-term planning. The framework operates through multimodal perception capabilities, integrating visual, textual, and contextual information from mobile environments to inform decision-making and tool selection processes.

## Key Results
- Achieves approximately 22% reduction in reasoning costs compared to baseline systems
- Demonstrates improved success rates across all task complexity levels (3.2%, 22.2%, and 22.2% improvements)
- Superior performance in handling complex, long-term planning tasks while maintaining operational efficiency

## Why This Works (Mechanism)
The framework's effectiveness stems from its dynamic team assembly approach that matches task complexity with appropriate expert specialization. By allowing each agent to independently formulate tools through code-combination methods, the system adapts to diverse task requirements without rigid tool pre-specification. The dual-layer planning mechanism separates concerns between high-level task distribution and low-level atomic operations, enabling efficient parallel processing while maintaining coherent task execution. The proactive working memory system enables agents to maintain context across extended task sequences, supporting long-term planning capabilities essential for complex mobile operations.

## Architecture Onboarding

Component Map:
MobileExperts Framework -> Multimodal Perception Module -> Dual-Layer Planning Engine -> Dynamic Agent Assembly -> Code-Combination Tool Generator -> Proactive Working Memory -> Task Execution Interface

Critical Path:
Task Input → Multimodal Analysis → Team Formation → Tool Formulation → Memory Initialization → Task Decomposition → Expert Execution → Result Aggregation

Design Tradeoffs:
The framework prioritizes flexibility and adaptability over computational efficiency by enabling dynamic tool generation and team assembly. This approach increases reasoning costs but provides superior handling of novel and complex tasks. The dual-layer planning introduces additional coordination overhead but enables better scalability for multi-expert scenarios. The proactive working memory system requires additional storage and processing but enables crucial long-term planning capabilities.

Failure Signatures:
- Team coordination breakdowns when agent communication latency exceeds thresholds
- Tool formulation failures when code combinations exceed complexity limits
- Memory interference when working memory capacity is insufficient for task complexity
- Planning mismatches when dual-layer coordination fails to align expert-level and team-level objectives

3 First Experiments:
1. **Team Assembly Validation**: Test dynamic agent selection accuracy by presenting tasks of varying complexity and measuring whether the correct expert combinations are chosen
2. **Tool Formulation Testing**: Evaluate code-combination tool generation by presenting novel task scenarios and measuring successful tool creation rates
3. **Memory System Performance**: Assess proactive working memory effectiveness by tracking context retention across multi-step task sequences
