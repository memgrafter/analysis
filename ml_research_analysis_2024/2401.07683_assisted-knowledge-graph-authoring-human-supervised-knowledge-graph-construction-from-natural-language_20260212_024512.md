# Assisted Knowledge Graph Authoring: Human-Supervised Knowledge Graph Construction from Natural Language

## Quick Facts
- arXiv ID: 2401.07683
- Source URL: https://arxiv.org/abs/2401.07683
- Authors: Marcel Gohsen; Benno Stein
- Reference count: 25
- Introduces WAKA, a web-based tool for creating and authoring knowledge graphs from unstructured text

## Executive Summary
This paper presents WAKA, a web-based tool for creating and authoring knowledge graphs from unstructured text with human supervision. The tool addresses the challenge of constructing domain-specific knowledge graphs by combining automatic knowledge graph construction with an intuitive interface that enables domain experts to supervise and correct the generated structures. The approach combines entity discovery, relation extraction, and knowledge fusion with Wikidata grounding, achieving recall values of 0.912 for named entity recognition, 0.765 for entity retrieval, and 0.778 for relation extraction.

## Method Summary
The core method involves an automatic knowledge graph construction algorithm that processes unstructured text through multiple stages: entity discovery using named entity recognition, relation extraction to identify connections between entities, and knowledge fusion with Wikidata to ground entities and relationships. The system then presents the automatically generated knowledge graph through WAKA's web interface, where domain experts can supervise, validate, and correct the generated structures. This human-in-the-loop approach aims to improve the quality and accuracy of knowledge graphs, particularly for domain-specific applications where fully automatic approaches may fall short.

## Key Results
- Recall values of 0.912 for named entity recognition, 0.765 for entity retrieval, and 0.778 for relation extraction
- The tool successfully enables domain experts to supervise and correct automatically generated knowledge graphs
- Demonstrates the importance of human supervision in knowledge graph construction due to the complexity of the task

## Why This Works (Mechanism)
The effectiveness of WAKA stems from its hybrid approach that combines automated knowledge graph construction with human supervision. By leveraging existing NLP techniques for entity recognition and relation extraction while providing an intuitive interface for domain experts to validate and correct outputs, the system balances efficiency with accuracy. The integration with Wikidata provides a reliable grounding mechanism, while the supervision layer addresses the limitations of fully automatic approaches that often struggle with domain-specific terminology and complex relationships.

## Foundational Learning
- Named Entity Recognition (NER): Identifies and classifies named entities in text (people, organizations, locations, etc.)
  - Why needed: Forms the foundation for identifying the nodes in knowledge graphs
  - Quick check: Test on sample medical text with domain-specific terminology

- Relation Extraction: Identifies relationships between entities in text
  - Why needed: Establishes the edges that connect nodes in knowledge graphs
  - Quick check: Verify extraction accuracy on sentences with explicit and implicit relationships

- Knowledge Fusion with Wikidata: Grounds entities to a reliable knowledge base
  - Why needed: Ensures consistency and provides semantic context for entities
  - Quick check: Validate entity matching accuracy against manually curated ground truth

## Architecture Onboarding
**Component Map:** Text Input -> NER -> Relation Extraction -> Wikidata Grounding -> Knowledge Graph -> WAKA Interface -> Human Supervision -> Updated Knowledge Graph

**Critical Path:** The most critical path is from relation extraction through human supervision, as errors in relationship identification are most difficult to automatically correct and require expert judgment.

**Design Tradeoffs:** The system prioritizes recall over precision in automatic processing to ensure completeness, relying on human supervision to filter and correct errors. This trades computational efficiency for higher quality outputs.

**Failure Signatures:** Common failure modes include misidentifying domain-specific entities, missing implicit relationships, and incorrect Wikidata mappings for specialized terminology. The interface should flag these areas for human review.

**First Experiments:** 1) Process a sample medical research abstract to test domain-specific entity recognition, 2) Extract relationships from a complex sentence with nested clauses, 3) Validate Wikidata grounding accuracy for specialized scientific terms.

## Limitations
- Performance metrics focus on recall without providing corresponding precision values
- Limited evaluation of the tool's effectiveness in actual domain-specific scenarios, particularly in medicine
- No information provided about dataset size, domain specificity, or text complexity used for testing

## Confidence
The confidence in the core claims is **Medium**. While the methodology for human-supervised knowledge graph construction is well-defined and the tool's interface is described in detail, the evaluation metrics are incomplete and lack precision-recall tradeoffs.
