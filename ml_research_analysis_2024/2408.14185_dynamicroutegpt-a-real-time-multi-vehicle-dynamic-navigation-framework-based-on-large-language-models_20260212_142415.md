# DynamicRouteGPT: A Real-Time Multi-Vehicle Dynamic Navigation Framework Based on Large Language Models

## Quick Facts
- arXiv ID: 2408.14185
- Source URL: https://arxiv.org/abs/2408.14185
- Authors: Ziai Zhou; Bin Zhou; Hao Liu
- Reference count: 40
- This paper presents DynamicRouteGPT, a real-time multi-vehicle dynamic navigation framework based on large language models for complex traffic environments. The framework integrates Markov chains, Bayesian inference, and Llama3 8B to provide efficient path planning solutions, dynamically adjusting to traffic scenarios and driver preferences without pre-training. Experimental results show state-of-the-art performance in travel time reduction across various road networks, with average waiting time decreasing by 64.1% compared to XRouting and 81.8% compared to DQN in Manhattan scenarios.

## Executive Summary
DynamicRouteGPT introduces a novel real-time multi-vehicle navigation framework that leverages large language models for dynamic path planning in complex traffic environments. The system combines global optimality from Dijkstra's algorithm with real-time decision-making through LLM-driven Bayesian inference, achieving significant reductions in travel and waiting times. By integrating Markov chains for traffic state prediction and LoRA fine-tuning for efficient model adaptation, the framework demonstrates strong generalization capabilities across different traffic conditions and vehicle penetration rates.

## Method Summary
DynamicRouteGPT employs a four-module architecture: Real-Time Information Acquisition (using Markov chains and SUMO integration), Alternative Path Generation (Dijkstra + k-shortest paths), GPT Decision-Making (LLM + Bayesian inference), and Dynamic Path Adjustment (SUMO vehicle control loop). The framework first computes a globally optimal baseline path using Dijkstra's algorithm, then at each intersection, the Llama3 8B model evaluates multiple candidate paths based on real-time traffic data and updates path probabilities via Bayesian inference to select the best local continuation. LoRA fine-tuning is applied to adapt the base LLM for navigation tasks without full retraining.

## Key Results
- Average waiting time decreased by 64.1% compared to XRouting and 81.8% compared to DQN in Manhattan scenarios
- Framework achieves state-of-the-art performance in travel time reduction across various road networks
- Demonstrates strong generalization capability across different traffic conditions and vehicle penetration rates up to 15%

- Design tradeoffs:
  - Global optimality vs. real-time responsiveness: Dijkstra provides a stable global scaffold but may be outdated; real-time LLM decisions adapt but risk local traps.
  - Computational cost vs. adaptability: LoRA fine-tuning balances model specialization with efficiency; full fine-tuning would be more costly.
  - Model complexity vs. interpretability: LLM-based decisions offer explainability through causal graphs but add opacity compared to classical planners.

- Failure signatures:
  - Excessive rerouting or oscillation at intersections may indicate poor Bayesian weighting or stale Markov state estimates.
  - Consistently worse travel times than Dijkstra alone may suggest LLM decisions are overfitting local conditions at the expense of global coherence.
  - Simulation crashes or out-of-bounds vehicle states may indicate integration errors between LLM decisions and SUMO control.

## Limitations
- Limited real-world validation: All experiments conducted in SUMO simulations; performance in live traffic remains untested
- Computational overhead concerns: Real-time LLM inference with LoRA fine-tuning at each intersection may not scale efficiently to city-wide deployments
- Black-box decision factors: While causal graphs provide explanations, the LLM's internal reasoning process for path selection is not fully transparent
