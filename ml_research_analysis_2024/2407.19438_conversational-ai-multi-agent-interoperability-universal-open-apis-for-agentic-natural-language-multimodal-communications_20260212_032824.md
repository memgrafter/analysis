# Conversational AI Multi-Agent Interoperability, Universal Open APIs for Agentic Natural Language Multimodal Communications

## Quick Facts

## Executive Summary
This paper presents a framework for enabling interoperability among diverse conversational AI agents using Universal Open APIs based on natural language. The approach allows agents to collaborate seamlessly, regardless of their underlying technology (e.g., LLMs, non-AI systems) or communication channels (voice, video, text). A key innovation is the use of natural language-based APIs for agent discovery and service manifest publication, facilitating efficient agent interactions. The framework is demonstrated through state diagrams and two use cases, including a "Smart Errands" scenario and a "Smart Library" service for Estonian citizens. The results show that the system can effectively coordinate tasks among multiple agents, providing specialized support independently of specific AI technologies. Future enhancements include improved security, bias mitigation, and multi-party conversation support.

## Method Summary
The framework implements Universal Open APIs based on natural language to enable interoperability among conversational AI agents. It uses a JSON-based Conversation Envelope structure with schema version 0.9.2 for message exchange, four types of Dialog Events (Utterance, Whisper, Invite, Bye) for agent communication, and Assistant Manifests for service discovery and capability description. The method involves defining state transitions for agent interactions, implementing a discovery process for agent lookup through requestManifest and publishManifest events, and demonstrating the framework through practical use cases with specific agent interactions. The approach prioritizes loose coupling and technology independence over performance optimization, allowing agents to work with diverse AI technologies without custom integration code.

## Key Results

- Natural language-based discovery and manifest publication facilitate efficient agent interactions
- State diagrams based on OVON specifications map agent interactions and status transitions

## Architecture Onboarding

- Component map: Conversation Envelope API (message exchange) -> Discovery Framework (agent lookup and manifest publication) -> State Management (tracking agent interactions)
- Critical path: Agent A invites Agent B to conversation -> Agent B accepts invitation and enters READY state -> Agent A sends utterance/whisper to Agent B -> Agent B processes request and sends response -> Agents exchange bye events to terminate conversation
- Design tradeoffs: The framework prioritizes loose coupling and technology independence over performance optimization, meaning agents may experience higher latency due to natural language processing overhead but gain the ability to work with diverse AI technologies without custom integration code.
- Failure signatures: Common failure modes include agent discovery failures due to manifest parsing errors, state synchronization issues when agents have different interpretations of conversation context, timeout failures during agent lookup or response generation, and natural language ambiguity causing capability mismatch.
- First 3 experiments:
  1. Implement two simple text-based agents that exchange basic utterance and bye events to verify JSON envelope parsing and basic state transitions.
  2. Create a manifest publication and discovery scenario where one agent publishes capabilities and another successfully finds and invokes those capabilities.
  3. Build a multi-agent conversation with a human proxy agent to test the complete invitation, interaction, and termination flow across different agent technologies.

## Limitations

- The complexity of real-world multi-agent interactions may exceed the expressiveness of the current manifest system.
- The paper lacks quantitative performance metrics and does not address potential security vulnerabilities in the open API approach.
