# ReMatching Dynamic Reconstruction Flow

## Quick Facts
- arXiv ID: 2411.00705
- Source URL: https://arxiv.org/abs/2411.00705
- Reference count: 32
- Key outcome: Achieves up to 42.40 PSNR on D-NeRF, outperforming state-of-the-art baselines

## Executive Summary
This paper introduces ReMatching, a framework for incorporating deformation priors into dynamic reconstruction models through a flow-matching approach. The method formulates the integration of velocity-field-based priors as an analytically solvable optimization problem, enabling models to satisfy both reconstruction objectives and prior regularization simultaneously. Experiments on synthetic and real-world datasets demonstrate significant improvements in reconstruction accuracy, particularly around moving parts, with fewer artifacts and distortions.

## Method Summary
ReMatching integrates deformation priors into dynamic reconstruction by formulating the problem as a flow-matching optimization. The framework assumes a time-dependent reconstruction function and projects it onto a velocity-field prior class using an analytical solution. The method supports simulation-free flow training and can combine multiple simpler priors into more complex classes through learnable weights. The ReMatching loss is added to the reconstruction objective with a weighting factor (λ=0.001), enabling seamless integration into existing dynamic reconstruction pipelines.

## Key Results
- Achieves 42.40 PSNR on D-NeRF dataset, outperforming D3G (41.30 PSNR) and K-Planes (32.50 PSNR)
- Demonstrates improved reconstruction quality around moving parts with fewer artifacts
- Shows applicability to real-world scenes through HyperNeRF experiments
- Enables incorporation of priors for time-dependent rendered images with plausible reconstructions

## Architecture Onboarding

- Component map: Time-dependent reconstruction function -> Velocity-field prior class -> Matching optimization problem -> Flow matching loss -> ReMatching loss
- Critical path: Evaluation of ReMatching loss involving solving matching optimization problem and computing flow matching loss, must be efficient to avoid slowing training
- Design tradeoffs: Choice of prior class involves tradeoff between expressivity and tractability; more complex priors capture real-world deformations better but are harder to optimize; adaptive-combination scheme allows more complex priors but introduces additional learnable parameters
- Failure signatures: Ineffective ReMatching loss shows unrealistic distortions or artifacts in reconstructions, especially moving parts; unstable matching problem optimization causes training convergence failure
- First 3 experiments:
  1. Apply ReMatching framework with simple prior class (directional restriction) to synthetic dataset (D-NeRF) and compare reconstructions to baseline without priors
  2. Experiment with different prior classes (rigid, volume-preserving) to see effects on different deformation types
  3. Test adaptive-combination scheme on piecewise-deforming scenes to see if it effectively combines simpler priors for different parts
