# Textured Gaussians for Enhanced 3D Scene Appearance Modeling

## Quick Facts
- arXiv ID: 2411.18625
- Source URL: https://arxiv.org/abs/2411.18625
- Reference count: 40
- Primary result: Achieves up to 2 dB PSNR improvement over 3DGS using only 1% of Gaussians

## Executive Summary
This paper addresses the fundamental limitation of 3D Gaussian Splatting where each Gaussian can only represent a single color and simple ellipsoidal shape. The authors propose augmenting each Gaussian with alpha, RGB, or RGBA texture maps, enabling spatially varying colors and opacities. This approach allows each Gaussian to represent complex shapes beyond simple ellipsoids and capture high-frequency texture details. Experiments demonstrate significant improvements in image quality using the same number of Gaussians as 3DGS, and even greater improvements when using only 1% of the default Gaussian count.

## Method Summary
The method enhances 3D Gaussian Splatting by attaching per-Gaussian texture maps to each Gaussian primitive. Each Gaussian is augmented with texture maps (alpha-only, RGB, or RGBA) that are sampled based on the intersection point of rays with the Gaussian's bounding plane. The intersection point is mapped to UV coordinates, and bilinear interpolation is used to sample the texture map. A two-stage optimization process is employed: first optimizing a baseline 3DGS model, then jointly optimizing Gaussian attributes with texture maps for an additional 30,000 iterations. The approach maintains the efficiency of 3DGS while significantly improving expressivity through spatially varying appearance.

## Key Results
- Alpha-only texture maps significantly improve Gaussian expressivity, enabling complex shapes through spatially varying opacity
- Full RGBA texture maps achieve maximum expressivity by combining spatially varying color and opacity
- Using only 1% of default Gaussians achieves nearly 2 dB PSNR improvement over 3DGS
- Alpha-only variant achieves comparable performance to full RGBA while using one-third the model size

## Why This Works (Mechanism)

### Mechanism 1
- Claim: Alpha-only texture maps significantly improve Gaussian expressivity by enabling spatially varying opacity and alpha composition.
- Mechanism: Each Gaussian's shape is no longer limited to a simple ellipsoid; the alpha channel of the texture map modulates opacity across the Gaussian's extent. This allows complex shapes to be represented and enables alpha-composited colors to capture high-frequency textures.
- Core assumption: Spatially varying opacity across a Gaussian can be encoded efficiently in a per-Gaussian texture map without requiring a global texture map.
- Evidence anchors:
  - [abstract] "Notably, alpha-only texture maps significantly improve Gaussian expressivity, while further augmenting with RGB texture maps achieve maximum expressivity."
  - [section] "Interestingly, using alpha-only textures already outperforms 3DGS∗, and our RGB textures model despite being one-third the size, striking a better balance between performance and model size. This is because alpha-textured Gaussians can represent complex shapes through spatially varying opacity and reconstruct high-frequency textures through spatially varying alpha composition."
- Break condition: If the texture map resolution is too low to capture the spatial variation in opacity, or if the per-Gaussian texture maps become too memory-intensive relative to model performance gains.

### Mechanism 2
- Claim: RGB texture maps achieve maximum expressivity by adding spatially varying color information to the Gaussian.
- Mechanism: While alpha textures allow for complex shapes, RGB textures provide spatially varying color information beyond the low-frequency colors captured by spherical harmonics. This combination allows each Gaussian to represent high-frequency color variations across its extent.
- Core assumption: The combination of spatially varying color (RGB) and spatially varying opacity (alpha) provides more expressivity than either alone.
- Evidence anchors:
  - [abstract] "Interestingly, alpha-only texture maps significantly improve Gaussian expressivity, while further augmenting with RGB texture maps achieve maximum expressivity."
  - [section] "With RGB textures, base color reconstructs low-frequency textures and texture maps reconstruct high-frequency elements."
- Break condition: If the additional expressivity from RGB textures does not translate to improved image quality, or if the model size becomes prohibitive compared to the performance gain.

### Mechanism 3
- Claim: There is a sweet spot between texture map resolution and number of Gaussians that achieves optimal performance.
- Mechanism: Increasing texture map resolution allows for finer detail reconstruction but reduces the number of Gaussians that can be used given a fixed model size. Conversely, more Gaussians with lower resolution textures can better capture geometry but may miss fine texture details.
- Core assumption: Model size is a fixed budget that can be allocated between Gaussian parameters and texture map channels.
- Evidence anchors:
  - [section] "We observe a sweet spot between texture map resolution and number of Gaussians that achieves the best image reconstruction quality. Maximizing texture map resolution corresponds to greatly reducing the number of Gaussians, which harms reconstruction of detailed geometric structures. Minimizing the texture map resolution, on the other hand, simply degenerates into 3DGS∗."
  - [section] "In Figure 9, we optimize alpha-only Textured Gaussians models of the same model size with different texture map resolutions and number of Gaussians, and show the trend of novel view synthesis performance on the Blender and DTU datasets."
- Break condition: If the optimal allocation point shifts significantly with different datasets or scene complexities, or if the model size budget changes dramatically.

## Foundational Learning

- Concept: Ray-Gaussian intersection and UV mapping
  - Why needed here: The method requires computing the intersection point of a ray from the camera to a pixel with each Gaussian, then mapping this point to UV coordinates to sample the texture map.
  - Quick check question: Given a Gaussian centered at μ with normal n, and a ray from camera origin o to pixel p, how do you compute the intersection point x and its corresponding UV coordinates?

- Concept: Alpha compositing
  - Why needed here: The final color of a pixel is computed by alpha-compositing the contributions from all Gaussians that the ray intersects, using their spatially varying colors and opacities.
  - Quick check question: Given the color c_i and alpha α_i for each Gaussian i, how do you compute the final pixel color using alpha compositing?

- Concept: Spherical harmonics for low-frequency color
  - Why needed here: Spherical harmonics coefficients provide a compact representation of view-dependent low-frequency color variations, complementing the high-frequency spatially varying colors from the texture maps.
  - Quick check question: What is the role of spherical harmonics coefficients in the Textured Gaussians model, and how do they differ from the texture map contribution?

## Architecture Onboarding

- Component map:
  - Ray casting -> Custom CUDA kernel for ray-Gaussian intersection
  - Texture mapping -> Custom CUDA kernel for UV mapping and bilinear interpolation of texture maps
  - Color computation -> Generalized Gaussian appearance model that combines SH-based base color and texture-mapped color
  - Alpha computation -> Generalized Gaussian appearance model that combines SH-based opacity and texture-mapped alpha
  - Rendering -> Modified 3DGS rendering pipeline with alpha compositing

- Critical path:
  1. For each pixel, cast a ray from the camera origin
  2. Intersect the ray with all Gaussians in the scene
  3. For each intersected Gaussian, compute the intersection point and UV coordinates
  4. Sample the texture map at the UV coordinates to get color and alpha values
  5. Compute the final color and alpha using the generalized Gaussian appearance model
  6. Alpha-composite the contributions from all intersected Gaussians

- Design tradeoffs:
  - Texture map resolution vs. number of Gaussians: Higher resolution allows finer detail but reduces the number of Gaussians for a fixed model size
  - Memory usage: Per-Gaussian texture maps increase memory requirements compared to 3DGS, but enable spatially varying appearance
  - Training complexity: Two-stage optimization (3DGS pretraining + texture map optimization) improves convergence but doubles training time

- Failure signatures:
  - Visual artifacts at grazing angles: May indicate issues with ray-Gaussian intersection or texture mapping near Gaussian boundaries
  - Blurry textures: Could indicate insufficient texture map resolution or improper UV mapping
  - Geometric inaccuracies: May suggest the need for more Gaussians or better optimization of Gaussian parameters

- First 3 experiments:
  1. Implement ray-Gaussian intersection and UV mapping kernels, verify correct texture sampling on a simple scene
  2. Integrate the generalized Gaussian appearance model with the 3DGS rendering pipeline, test on a scene with alpha textures only
  3. Add RGB texture support and compare performance with alpha-only and 3DGS baselines on a benchmark dataset

## Open Questions the Paper Calls Out

- Question: How does the optimal balance between texture map resolution and the number of Gaussians change across different scene types (e.g., object-level vs. scene-level datasets)?
- Question: Can Textured Gaussians be extended to model spatially varying specular color and other non-diffuse textures?
- Question: How would the integration of factorized representations like TensoRF or triplane improve the representation of high-dimensional textures in Textured Gaussians?

## Limitations
- The optimal balance between texture map resolution and Gaussian count is not theoretically derived and appears dataset-dependent
- The method assumes all textures are diffuse and lie on local surfaces, not modeling spatially varying specular colors
- Two-stage optimization adds significant complexity without clear theoretical justification

## Confidence

**High Confidence**: Claims about improved PSNR/SSIM/LPIPS metrics compared to baseline 3DGS, as these are directly measured and reproducible

**Medium Confidence**: Claims about alpha-only textures providing sufficient expressivity for most cases, as this is primarily based on ablation studies on specific datasets

**Low Confidence**: Claims about the method being a "drop-in replacement" for 3DGS, given the added complexity of texture map management and potential memory constraints

## Next Checks

1. **Cross-dataset generalization test**: Evaluate the proposed method on datasets with significantly different characteristics (e.g., indoor architectural scenes, outdoor natural scenes) to verify the claimed performance improvements are not dataset-specific.

2. **Memory-efficiency analysis**: Conduct a detailed analysis of memory usage as a function of texture map resolution and Gaussian count to identify practical limits and validate the claimed efficiency benefits of alpha-only textures.

3. **Failure case investigation**: Systematically identify and analyze failure modes (e.g., texture blurring, geometric inaccuracies, artifacts at grazing angles) to better understand the method's limitations and develop diagnostic tools for practitioners.
