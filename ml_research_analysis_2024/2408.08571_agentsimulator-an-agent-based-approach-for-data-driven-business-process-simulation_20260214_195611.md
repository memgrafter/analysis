# AgentSimulator: An Agent-based Approach for Data-driven Business Process Simulation

## Quick Facts
- **arXiv ID**: 2408.08571
- **Source URL**: https://arxiv.org/abs/2408.08571
- **Reference count**: 23
- **Primary result**: Achieves state-of-the-art simulation accuracy across nine datasets with significantly lower computation times than existing methods

## Executive Summary
This paper introduces AgentSimulator, a resource-first approach to business process simulation that discovers a multi-agent system from event logs. Unlike traditional control-flow-first approaches, AgentSimulator explicitly models distinct resource behaviors, interaction patterns, and decentralized decision-making through autonomous agents. The approach achieves state-of-the-art simulation accuracy across nine datasets, with significantly lower computation times than existing methods. AgentSimulator demonstrates superior performance in capturing control-flow patterns, temporal distributions, and cycle times, while maintaining high interpretability and adaptability to different process types through automated selection between orchestrated and autonomous handover configurations.

## Method Summary
AgentSimulator employs a resource-first methodology that discovers multi-agent systems from event logs. The approach models business processes by defining distinct resource types as autonomous agents with individual behaviors and decision-making capabilities. These agents interact through predefined interaction patterns to simulate process execution. The system automatically selects between orchestrated and autonomous handover configurations based on the characteristics of the input data, allowing it to adapt to different process types. This contrasts with traditional control-flow-first approaches that prioritize process structure over resource behavior modeling.

## Key Results
- Achieves state-of-the-art simulation accuracy across nine datasets
- Significantly lower computation times compared to existing methods
- Superior performance in capturing control-flow patterns, temporal distributions, and cycle times

## Why This Works (Mechanism)
AgentSimulator works by explicitly modeling resource behaviors and interactions rather than focusing primarily on control flow. This resource-first approach captures the decentralized decision-making and autonomous behaviors of real-world resources more accurately. By discovering a multi-agent system directly from event logs, the method preserves the natural variability and complexity of actual process execution. The automated configuration selection between orchestrated and autonomous handovers allows the system to adapt its simulation strategy to match the underlying process characteristics, optimizing both accuracy and performance.

## Architecture Onboarding

**Component Map**: Event Logs -> Log Analyzer -> Agent Discovery -> Configuration Selector -> Multi-agent System -> Simulation Engine -> Accuracy Metrics

**Critical Path**: Event logs flow through log analysis to discover agents and their interaction patterns, then configuration selection determines the handover strategy, and finally the multi-agent system executes simulations with measured accuracy.

**Design Tradeoffs**: Resource-first approach sacrifices some control-flow precision for more realistic resource behavior modeling, but gains significant computational efficiency and better captures real-world variability.

**Failure Signatures**: Poor accuracy may indicate insufficient event log detail, incorrect agent behavior modeling, or inappropriate configuration selection between orchestrated and autonomous modes.

## Limitations
- Evaluation relies on publicly available event logs which may not capture full real-world complexity
