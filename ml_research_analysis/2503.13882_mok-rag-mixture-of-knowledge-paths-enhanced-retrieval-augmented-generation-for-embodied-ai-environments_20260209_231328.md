# MoK-RAG: Mixture of Knowledge Paths Enhanced Retrieval-Augmented Generation for Embodied AI Environments

## Quick Facts
- **arXiv ID**: 2503.13882
- **Source URL**: https://arxiv.org/abs/2503.13882
- **Reference count**: 17
- **Primary result**: Multi-source RAG framework MoK-RAG reduces missing rates for main objects (9.52%) and paired objects (27.22%) in 3D scene generation

## Executive Summary
MoK-RAG addresses the Reply Missing problem in RAG systems by partitioning the LLM corpus into distinct sections, enabling retrieval from multiple specialized knowledge paths. This multi-source approach is particularly effective for embodied AI environments where comprehensive knowledge retrieval is critical. The framework demonstrates significant improvements in reducing missing rates for both main objects and paired objects in 3D scene generation tasks, while producing more realistic and diverse scenes compared to single-source methods.

## Method Summary
MoK-RAG is a novel multi-source RAG framework that enhances retrieval by partitioning the LLM corpus into distinct sections, allowing retrieval from multiple specialized knowledge paths. For 3D environment generation, MoK-RAG3D extends this approach by partitioning 3D assets based on a hierarchical knowledge tree structure. The system leverages CLIP score comparisons for automated evaluation and demonstrates superior performance in generating complete and diverse scenes compared to baseline methods.

## Key Results
- MoK-RAG3D reduces missing rate of main objects by 9.52% compared to single-source methods
- MoK-RAG3D reduces missing rate of paired objects by 27.22% compared to single-source methods
- Outperforms baseline methods in CLIP score comparisons across residential scene types

## Why This Works (Mechanism)
The multi-source retrieval approach addresses the Reply Missing problem by ensuring comprehensive coverage of knowledge through specialized partitions. By maintaining distinct knowledge paths rather than relying on a single monolithic corpus, the system can retrieve relevant information from multiple perspectives, reducing the likelihood of missing critical objects or relationships in generated scenes.

## Architecture Onboarding

**Component map**: User query -> Multiple knowledge path retrievers -> Knowledge fusion module -> LLM generation module -> 3D scene output

**Critical path**: Query processing -> Parallel retrieval from specialized knowledge paths -> Knowledge path fusion -> Context-aware generation

**Design tradeoffs**: The multi-source approach trades increased computational overhead for improved retrieval completeness and scene quality. The hierarchical partitioning balances specificity with coverage breadth.

**Failure signatures**: Missing critical objects when specialized knowledge paths fail to overlap appropriately; Reduced diversity when fusion mechanism overly prioritizes certain knowledge paths
