# AccLLM: Accelerating Long-Context LLM Inference Via Algorithm-Hardware Co-Design

## Quick Facts
- **arXiv ID**: 2505.03745
- **Source URL**: https://arxiv.org/abs/2505.03745
- **Reference count**: 40
- **Key result**: 4.07× energy efficiency and 2.98× throughput vs FlightLLM on Xilinx Alveo U280

## Executive Summary
AccLLM introduces an algorithm-hardware co-design framework that achieves efficient long-context LLM inference on FPGAs by combining aggressive compression techniques with a dedicated hardware accelerator. The approach uses 2:4 semi-structured pruning, Λ-shaped attention, and W2A8KV4 quantization to drastically reduce model size, memory overhead, and bandwidth demands while maintaining accuracy. A reconfigurable computing engine with flexible DSP packing is designed to fully exploit these optimizations, supporting mixed-precision operations and sparse patterns. Experiments demonstrate 1.53 GB Llama-2-7B model size with preserved performance on WikiText-103/2, achieving 4.07× energy efficiency and 2.98× throughput compared to state-of-the-art FlightLLM.

## Method Summary
AccLLM combines aggressive compression with specialized hardware acceleration for long-context LLM inference. Algorithmically, it applies 2:4 semi-structured pruning to linear layers, uses W2A8KV4 quantization (2-bit weights, 8-bit activations, 4-bit KV cache), and implements Λ-shaped attention that retains only initial and recent tokens. Accuracy recovery is achieved through LoRA fine-tuning on quantized weights. The hardware consists of a reconfigurable computing engine with T processing tiles, each containing M PE blocks with R precision-scalable multipliers, supporting both matrix-matrix (prefill) and vector-matrix (decode) operations with flexible DSP packing for mixed-precision sparse computations.

## Key Results
- 4.07× energy efficiency and 2.98× throughput compared to FlightLLM on Xilinx Alveo U280
- Llama-2-7B model compressed to 1.53 GB while preserving WikiText-103/2 performance
- KV cache reduced from 3.5 GB to 0.25 GB (93% reduction) using Λ-attention + 4-bit quantization
- 1.91× speedup from W2 quantization, 1.28× from flexible DSP packing

## Why This Works (Mechanism)

### Mechanism 1: W2A8KV4 Quantization with LoRA Recovery
Aggressive 2-bit weight quantization combined with 4-bit KV cache compression reduces memory bandwidth bottlenecks while LoRA fine-tuning recovers accuracy degradation. Group-wise quantization with learnable weight clipping compresses weights to 2-bit. After pruning+quantization causes perplexity degradation (16.695→7.408 on WikiText-2), LoRA adds low-rank trainable matrices (A, B) that are later quantized to 8-bit. The final computation approximates original weights as: XW ≈ XfWQ + XAQ BQ^T. This adds only ~3% memory overhead while recovering near-baseline performance.

### Mechanism 2: Reconfigurable Computing Engine (RCE) with Flexible DSP Packing
A single hardware architecture efficiently handles MM/VM operations, sparse/dense patterns, and mixed-precision through runtime reconfiguration. The RCE contains T processing tiles, each with M PE blocks containing R precision-scalable multipliers. It reconfigures between: (1) MM mode for prefill—broadcast weights across tiles, accumulate partial sums within blocks; (2) VM mode for decode—parallel output channel processing across all tiles. The flexible DSP packing exploits DSP48E2's (A+D)×B structure to execute multiple operations per DSP: 2× (8-bit activation × 8-bit weight) or 4× (8-bit input × 2-bit pruned weight). A sparse selector uses multiplexers to bypass zero weights from 2:4 pruning.

### Mechanism 3: Λ-Shaped Attention for Long-Context KV Cache Reduction
Retaining KV states for only initial "attention sink" tokens and recent window tokens maintains long-context coherence while reducing KV cache memory by 71.4% (3.5GB→1GB for 7K tokens). Standard attention requires O(sequence_length) KV cache growth. Λ-shaped attention observes that: (1) initial tokens act as "attention sinks" with disproportionately high attention scores, (2) local context (recent tokens) dominates most attention patterns. The implementation retains 4 initial tokens + 2044 recent tokens (configurable), dropping intermediate history. Combined with 4-bit KV quantization, total reduction reaches ~93% (3.5GB→0.25GB).

## Foundational Learning

- **Concept: LLM Inference Stages (Prefill vs. Decode)**
  - Why needed: The entire accelerator design hinges on different optimization strategies for each stage. Prefill is compute-bound (matrix-matrix), decode is memory-bound (vector-matrix with KV cache).
  - Quick check: If you observed 90% compute utilization during prefill but only 10% during decode on a baseline implementation, which stage would you target for bandwidth optimization vs. compute optimization?

- **Concept: N:M Semi-Structured Sparsity**
  - Why needed: 2:4 pruning is the backbone of the compression strategy. Understanding that exactly 2 out of every 4 elements must be zero enables hardware-friendly sparse selector design.
  - Quick check: Given a weight tensor of shape [4096, 4096] with 2:4 sparsity applied column-wise, how many non-zero elements remain, and what is the theoretical speedup if sparse selection has zero overhead?

- **Concept: DSP48E2 Architecture and Packing**
  - Why needed: The flexible DSP packing strategy is a key hardware efficiency driver. Understanding the (A+D)×B multiplier structure explains why 4 multiplications can fit in one DSP for 8×2-bit operations but only 2 for 8×8-bit.
  - Quick check: When packing two 8-bit activations and two 2-bit weights into one DSP48E2, why must the sparse selector operate before packing, and what happens if the two inputs require different sparsity masks?
